# 从 SPH 到 CADET：一份研究叙事

> **CADET** = **C**ontrol-**A**llocation-**D**irected **E**xploration of **T**riggers（控制分配导向的触发探索）

**主题。** 系统性地发现无人机飞控程序中*可行但不安全的飞手输入*——即合法、受速率限制的人工摇杆序列，却会把一个配置正确的飞控逼到违反其飞行模式安全契约——并把它们最小化为人类可读的触发机动。

**谱系。** 本工作是 RouthSearch（ISSTA 2025）的后继，后者用一个原则性的稳定性判据（Routh–Hurwitz）来结构化对 PID *参数*空间的搜索。我们固定 PID 增益、传感器、固件、混控器（mixer）与任务，转而搜索*飞手输入*空间。门类是相同的：原则性判据 + 结构化搜索 + 离线鲁棒度 oracle，在 PX4/ArduPilot 上评估。

本文是一份诚实的记录：我们最终得到的方法——**CADET**——是如何从一连串被证伪的假设中浮现出来的。它的写法力求让每一步转折处的推理都清晰可读，包括那些死胡同，因为正是这些死胡同界定并证成了最终的方法。

---

## 0. 我们从未改动的设定

在下文每一个阶段中，实验框架都保持固定（这正是让那些负面结果可信的原因）：

- **输入参数化。** 飞手输入是一个 40 维向量：10 个时间窗口 × 0.5 s × 4 个通道（roll, pitch, yaw, throttle），覆盖 5 s 时域，随后接 8 s 回中尾段。所有被测属性都是*回中后*的——摇杆回到中位*之后*的漂移 / 速度。
- **属性（STL 契约）。** `post_neutral_xy_drift`（≤ 2.0 m）、`post_neutral_alt_drift`（≤ 1.0 m）、`post_neutral_xy_velocity`（≤ 1.0 m/s）。鲁棒度 ρ = 阈值 − 最大偏差；ρ > 0 安全，ρ < 0 违例。
- **探针范围。** 全程以 PX4 `px4_position`（POSCTL）、seed 0 作为单种子探针。多种子 / 多场景复现是任何论文主张的前提，而非我们已经完成的步骤。
- **噪声纪律。** 仿真器是随机的。后期阶段所报告的每一个鲁棒度值都是 J=5 的重复（跑 5 次），"鲁棒"意味着跨过 2σ 带（鲁棒违例需 `ρ_mean + 2·ρ_std < 0`；鲁棒安全需 `ρ_mean − 2·ρ_std > 0`）。这套纪律是在早期测量被噪声污染后才引入的，也是整个项目里最重要的单一方法学选择：它让我们在早期阶段拒绝*假阴性*、在后期阶段拒绝*假阳性*。

## 1. 第一幕 —— SPH，以及平台的发现

**假设（支撑持续性假设，Support Persistence Hypothesis, SPH）。** 鲁棒度梯度的*支撑*——即真正能让 ρ 移动的那些输入维度——在全局上是稀疏的，并且沿搜索路径*持续*。若为真，你就能一次性学到支撑，再复用它来做查询高效的搜索。

**我们的发现。**

1. **安全区域并不稀疏——它是一个平台。** 在回中输入周围，top-4 梯度质量仅 ≈ 0.23–0.30（"稀疏"需要 ≥ 0.5），有效稀疏度 ≈ 22–29（共 40 维；"稀疏"需要 ≤ 15）。诊断很直白：所测得的"梯度"几乎全是噪声。噪声 L1 ≈ 282× 名义逐步标准差；有效稀疏度 ≈ 25 恰恰是 40 维纯噪声所产生的值。多数属性是**鲁棒度平台**——ρ 平坦，梯度与噪声无法区分。有一个属性（`xy_velocity`）是一个浅而弥散的碗，但仍无任何低维结构。
2. **持续性失败。** 沿一条路径，"top"维度在相邻步之间的重叠与随机基线相当（质量重叠 ≈ 0.375 vs. 随机 0.40；Jaccard ≈ 0.28 vs. 随机 0.25）。梯度下降并不下降——鲁棒度有时反而*上升*。

**SPH 被证伪。** 但这次失败带来了第一个真正的洞见，并重塑了整个项目：

> **平台不是一个需要隐藏的负面结果——它是方法的第一幕。** 平台意味着你*无法*用梯度下降逃出安全区域；梯度指向虚无。任何结构都必然存在于*违例边界附近*，也就是说，只有在一次结构无关的搜索已经付出到达边界的代价之后才会出现。因此正确的架构是两段式：先不依赖梯度地到达边界，再在那里利用结构。

## 2. 第二幕 —— MarginSearch，以及三个承重的赌注

**重构。** 我们放弃了"全局稀疏支撑复用"，转而采用 **MarginSearch**：刻画低维的*违例边界*，跨条件热启动，并提取最小触发。理论锚点是*可行性理论*（viability theory；Nagumo 障碍 / HJ 可达性）：它许诺边界*存在*、是*低维*的（余维 1）、随条件*连续*移动、并在*模式切换处断裂*——这些都是结构，却没有一项能在黑盒飞控上计算出来。

三个假设支撑着这个方法。我们三个都做了检验。

### 2.1 H1 —— 边界是一个低维流形吗？

第一次测量把边界处的有效稀疏度定在约 21–26——而非我们想要的 ≤ 8。但边界后来被发现是一个*高噪声区域*（逐属性噪声 std 从平台到边界跳升约 50×），所以这是不确定（inconclusive），而非一个干净的负面结果。我们用大探针步长（δ = 0.2）重跑，把信号抬到噪声之上（≈ 7× SNR）。结果很锐利，也是此后一切的经验种子：

> **边界法向是*通道各向异性*的。** 它在**控制通道轴上稀疏**——roll + pitch 承载了 86% 的方向敏感度质量（通道参与比 2.62 / 4）——但在**时间轴上稠密**（窗口参与比 7.62 / 10，是一段连续的后段窗口带）。活跃集是一条约 12 元胞的 通道×时间 带，有效稀疏度 ≈ 21。

因此 H1 的*强*形式（可行性故事所预测的少数 cell-sparse "活跃约束"）是**假的**。但一个*真实*的结构幸存了下来：敏感度坍缩到了少数几个有物理意义的通道上（横向速度 ← roll/pitch 倾转——正是有文档记录的控制路径），并且这一点在不同条件下稳定。一个针对 `alt_drift` 的对照实验（它即便远离自身边界也会稀疏化）暗示这种集中可能是*幅值驱动*的、而非*边界局部*的，但在单条搜索射线上，幅值与边界邻近度是混杂的，所以那个问题仍**悬而未决**。

**H1 裁决：** cell-sparse / 可行性形式已死；通道各向异性是真实的，也是唯一经受住干净检验而幸存下来的结构性资产。

### 2.2 H2 —— 边界能跨条件热启动吗？

这是*效率*故事的成败关键。我们在同一工况内收紧速度契约（v_max ∈ {1.0, 0.9, 0.8}），追问：廉价地求解一个条件的边界，能否热启动其邻近条件。

**它失败了。** 对三个基线，热启动加速为 1.01–1.65×——低于 2× 的门槛。连续性前提其实*成立*（边界移动很小；roll+pitch 仍是活跃通道），但它没能转化为查询节省。原因比"热启动很弱"更深：

> **违例很便宜就能找到。** 纯均匀随机在约 50 次查询内就定位到了一个边界。没有昂贵的冷启动搜索可供摊销，所以热启动什么也没省下。雪上加霜的是，边界噪声定位税（穿过 ρ-std ≈ 0.04 的 J=5 包夹）冷暖一视同仁地都要交，并且淹没了任何距离上的优势。

**H2 裁决：** "靠摊销获得效率"的论点已死。而"违例很便宜就能找到"这个观察，在很久之后成了整个项目的转捩点。

### 2.3 H3 —— 模式切换处的不连续是 bug 吗？

最后一个未检验的假设，也是最有特色的潜在头条。我们搭了一个 `px4_transition` 场景（在参数化的切换时刻 POSCTL → AUTO_LOITER），验证了切换确实触发（顺带修了一个模式指令编码 bug），并搜索那些*在纯 POSCTL 下安全*却*在交接发生时违例*的输入——配以严格的 2σ 门限来拒绝噪声造成的伪跨界。

**它失败了。** 零个鲁棒的、由切换导致的违例。交接是干净的：切换只引入一个小而良好阻尼的瞬态，从不把一个安全输入推入违例，而 AUTO_LOITER 是一个*镇定*目标（它制止运动）。高幅值候选在不切换时就已经违例了（是坏输入，不是切换 bug）；温和候选无论切不切都安全。我们测的是最干净的那种"交接故障 = bug"形式，而 PX4 通过了。

**H3 裁决：** 切换交接在此不是 bug 来源（至少对 POSCTL → LOITER 而言）。

## 3. 清算

至此，六个假设已经倒下：全局 SPH、中心 SPH、持续性、强 H1、H2、H3。每一个把贡献框定为**效率**、或**可被利用以加速搜索的结构**、或**切换 bug** 的假设都死了。诱惑在于退守到一篇纯刻画论文（"这就是为什么在这个系统上证伪很难"）。诚实的张力是真实存在的：一篇只做刻画的论文，达不到 ISSTA 级别会场对*方法*贡献的要求。

出路不是又一个关于*结构*的假设。而是注意到：**度量指标从头到尾都搞错了。**

---

## 4. 转向 —— 该在什么上追求高效

数据里早已存在的两个观察，重构了一切：

1. **"找任意违例"是错误的目标。** 随机在约 50 次查询内就能找到违例——但*哪一种*违例？当我们去看时，随机的违例压倒性地是**平凡的**：饱和、满杆、"你指令了一个极端机动"。构成真正飞控 *bug* 的违例是**非平凡的**：内部（不贴着摇杆限位）、非饱和，并且——最强的形式——*回中*（飞手松手、飞行器却仍漂移越过界限）。这些有 bug 价值的违例是输入空间里的一层**薄壳**，随机很少落在上面。
2. **通道各向异性还活着——只是用途不同。** 它从来不是可行性故事想要的 cell 级稀疏，但在*通道轴*上它是真实、稳定、且扎根于控制分配的（横向运动由 roll/pitch 驱动）。它对"更快地找到违例"毫无用处（H2 已盖棺）。它对**可靠地产生最小、可解释的触发**可能是决定性的——一个不同且更有价值的目标。

于是问题变成了：*我们能否高效且可靠地找到那些非平凡、温和、可解释的触发——即真正的 bug——它们是随机会错过、且标准工具无法恢复的？* 这就是**方向 A**，与此前六个赌注不同，它押在一个*活着*的度量上。

## 5. 第三幕 —— 两个站住了的实验

我们用两个预注册（pre-registered）的判别性实验来检验方向 A 的承重前提（阈值在看到结果之前固定，以防确认偏误——这一次的风险是*假阳性*，因为我们希望方向 A 成立）。

### 5.1 前提是否成立，通道导向是否有帮助？（三臂探针）

匹配预算，每臂 N = 80 个 J=5 点，在 `xy_velocity` 上：

- **臂 A —— 均匀随机。** 47 个鲁棒违例，**0 个内部**；最温和的 max|θ| = 0.577，中位 0.83。随机不断采到温和输入，但温和的那些是*安全*的——它从不产生一个温和的违例。**薄目标前提成立。**
- **臂 B —— 随机 + 内部包夹（通道无关）。** 找到了 7 个随机错过的内部违例，最温和 0.373——但**稠密**（支撑 27，四个通道全活跃）。它到达了温和区域，却产生了不可解释的输入。
- **臂 C —— 通道导向（roll+pitch）。** 18 个内部违例，支撑 **4–8**，**仅 roll/pitch**——干净、单/双通道、人类可读的触发（"一段短暂的半杆 pitch，然后松手"）。

诚实的评分：通道导向在*峰值幅值*上**并不**取胜（臂 B 那种稀疏铺开的输入有更低的 max|θ|；这一点我们认）。它取胜在**内部违例产出**（18 vs 7 vs 0）与**触发干净度**（支撑 4–8、两通道 vs. 支撑 27、四通道）。贡献是*干净可解释的触发*，而非*更温和*或*更快*。

### 5.2 通道导向是否*必要*，还是先找后删就能匹敌？（ddmin 检验）

显而易见的反驳：拿一个稠密的通道无关违例，跑标准的 delta-debugging（ddmin）去最小化它。那能否恢复出臂 C 的干净触发，从而让通道*导向*的搜索变得多余？我们给了 ddmin 它最好的机会——整通道移除、分组最小化、幅值约简、慷慨的预算——并做了比较。

**ddmin 无法匹敌。** 只有 4/10 个起点最小化到了干净触发；最终支撑中位仍停在 14.5（vs. 臂 C 的 6）；半数情况下 throttle/yaw 残留；每个干净触发的代价高出一个数量级。其*机制*才是真正的发现，也是本文的方法学核心：

> **ddmin 需要鲁棒性裕度才能移除元胞。** 一个深度违例（ρ ≈ −1.2）有裕度：你可以剥掉很多元胞仍保持鲁棒违例，所以 ddmin 能把它清干净。一个**温和 / 浅**的违例（ρ ≈ −0.05，紧贴边界）则*没有*裕度：移除任何元胞都会把 ρ 推回噪声带之外，所以 ddmin 卡在一个稠密输入上。而有 bug 价值的违例*恰恰就是温和的那些*。更根本地，**干净的最小触发与稠密违例是输入空间中的不同点——二者之间不存在一条保持鲁棒的删除路径相连。** 一个支撑为 8 的协同 roll/pitch 输入可以是*深度*违例的（我们观察到 ρ = −0.64）；ddmin 无法从一个稠密、浅的起点删除得到它。删除式最小化是错误的工具。**你必须直接搜索那个结构化的、通道约简的空间。**

这正是把该方法证成为一种*方法*、而非"找一个违例、再最小化"的依据。

## 6. CADET —— 它是什么，以及为什么叫这个名字

**CADET** 是所有幸存下来的东西的综合。

一句话概括：**CADET 找出最温和的合法摇杆机动，用最少的控制通道表达，能在飞手松手之后破坏一个飞行模式安全契约。**

### 命名的纠偏（一段诚实的小插曲）

这个方法最初叫 **ChannelSearch**——直接套 RouthSearch / MarginSearch 的 `XSearch` 壳，再把 X 填成"我们在哪儿搜"。但这个名字有两个毛病，它们其实是同一个毛病：它读起来既廉价、又像在硬蹭 RouthSearch。

诊断很简单，而且和本文其余部分的纠偏同理——**我们一开始命名的对象搞错了**。RouthSearch / MarginSearch 这个谱系的命名逻辑是**以判据命名**：Routh(–Hurwitz) 是一个有名有姓的稳定性判据，Margin 是稳定裕度。名字指向的是"用什么原理来结构化搜索"。而 "Channel" 指向的是搜索的*维度 / 空间*（在哪些通道上搜），不是判据本身。套了壳，却没套到壳的灵魂。

而这个方法真正的"Routh 对应物"，是**控制分配**——从控制*通道*到一个安全属性所约束的机体运动的那张映射。这是判据；通道导向只是它的机制。于是改名为：

> **CADET = Control-Allocation-Directed Exploration of Triggers**（控制分配导向的触发探索）。

这次改名做了两件事：(a) 它**以判据命名**，从而把方法诚实地放进 RouthSearch 的谱系里——是同源的判据结构化搜索，而非借用表面形式；(b) 展开式恰好编码了三根支柱——判据（control allocation）、机制（directed）、产物（triggers）。它沿驱动每个属性的*少数控制通道*导向证伪——违例边界的敏感度正集中在那里——并瞄准*内部*边界，以合成最小、回中的触发机动。

### 诚实的范畴（从每一个死胡同里带过来的）

- 它在找*任意*违例上**并不**更省查询（随机很便宜——H2）。
- 它**不**产生比稀疏铺开搜索更低峰值幅值的输入（臂 B）。
- 它**不**依赖可行性理论（H1 的 cell-sparse 预测失败了；锚点是*经验*上的通道各向异性 + 控制分配）。
- 模式切换交接**不**是它的 bug 来源（H3）。
- 在被复现之前，它的主张是**单种子**的。

它*确实*做到、且被两个实验支撑的：它**可靠且廉价地产生最小、可解释、单/双通道、回中的触发**，针对一类**全新 bug**（可行但不安全的飞手输入）——这些触发是随机搜索完全错过的，也是通道无关的证伪 + delta-debugging 无法可靠恢复的，因为它们无法通过最小化一个稠密违例而到达。

## 7. 这趟旅程教会我们什么（方法层面的反思）

- **负面结果只有干净了才有价值。** 2σ / J=5 的纪律让我们既扔掉了噪声驱动的"结构"（早期 SPH），也扔掉了噪声驱动的"违例"（H3、ddmin）。项目一半的价值在于*没有*去报告被污染的正面结果。
- **一个失败的假设往往孕育着下一个。** 平台（SPH 之死）成了两段式搜索的动机。"违例很便宜"（H2 之死）成了正确度量的种子。通道各向异性（H1 的部分之死）成了方法的核心。
- **先把度量搞对，再搞方法。** 六个假设败给了*错误的*目标（找任意违例的效率；可供加速的可利用结构）。同一个系统，在对照*正确的*目标（可靠地合成有 bug 价值的最小触发）来测量时，给出了一个干净、且有机制解释的正面结果。
- **在论文里认下失败的东西。** 幅值这个度量对我们不利；效率与切换的故事都死了。把这些放进评估与威胁——而不是藏起来——正是让那个幸存的主张可信的原因。

| 安全区域是鲁棒度平台；朴素的梯度 / 全局稀疏方法失败 | 已确立（单种子） |
