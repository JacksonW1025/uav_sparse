# 论文大纲（ISSTA 风格）：CADET

> **CADET** = **C**ontrol-**A**llocation-**D**irected **E**xploration of **T**riggers（控制分配导向的触发探索）
>
> **暂定标题。** *CADET：通过控制通道导向的证伪，发现无人机飞控程序中可行但不安全的飞手输入*

本大纲把研究叙事映射到一篇会议论文的结构上。每一节列出其目的、所承载的内容，以及——关键地——其支撑结果是 **[已有]**（单种子探针）、**[必做]**（投稿前必须跑出来）还是 **[诚实负面]**（作为定界而非作为胜绩写进论文的结果）。

## 摘要

飞手通过合法、受速率限制的摇杆动作操控多旋翼。一个正确的飞控理应在任何此类输入下都把飞行器保持在其飞行模式安全契约之内——然而某些可行的摇杆机动会把它推出边界，包括这样的机动：*飞手已经把摇杆回到中位，飞行器却仍然漂移越过限位*。我们形式化了这一 bug 类——**可行但不安全的飞手输入**——并提出 **CADET**，一种把它们找出来并最小化为人类可读触发机动的证伪方法。CADET 建立在对飞控鲁棒性地形的刻画之上：在飞行器安全处它是*平坦*的，在违例附近是*通道各向异性*的（敏感度集中在驱动受约束运动的少数控制通道上）。我们表明，有 bug 价值的违例是温和、内部的输入，均匀随机搜索完全错过它们；并且——由于最小触发无法通过从稠密违例中删除元胞而到达——标准的证伪 + 最小化无法可靠地恢复它们。通过沿属性的控制通道导向搜索，CADET 直接合成出最小的、单/双通道的、回中的触发，并在 PX4 [及 ArduPilot] 中复现出真实 bug。*（数字来自单种子探针；多种子/跨平台结果待补。）*

## 1. 引言  **[承载贡献]**

**钩子 / 动机。** 一个具体、直观的 bug：一段短暂的半杆 pitch 输入，飞手把摇杆居中，多旋翼却继续平移越过其位置/速度契约。合法输入、正确配置、本该由飞控处理——而它没有处理。

**空白。** 既有的机器人飞行器测试 fuzz 的是*配置参数*、*GCS/任务指令*或*环境条件*（如 PGFuzz、RVFuzzer）。把连续的**人工摇杆时序**作为证伪目标基本上无人探索，尽管它是人类可达的最直接输入。CPS 证伪（S-TaLiRo 及其后继）针对通用输入信号、用鲁棒度下降来做，并不以产生*最小、人类可读、模式契约*的触发为目标。

**为何困难（机制预告）。** 两个障碍，均为经验性的：(i) 在飞行器安全处，鲁棒性是一个*平台*——没有梯度可以沿着逃出安全区域；(ii) 有 bug 价值的违例是*温和、内部*的输入，构成一层薄壳——随机采样会过冲进饱和（平凡）违例并错过它们，而最小化一个稠密违例无法到达一个温和干净违例，因为二者不被任何保持鲁棒性的删除路径连接。

**洞见 / 方法。** 在违例附近，鲁棒度敏感度是*通道各向异性*的：它集中在驱动受约束运动的少数控制通道上（横向速度 ← roll/pitch），与控制分配相匹配。CADET 把这一点作为搜索判据——沿相关通道导向证伪、瞄准内部边界——以直接合成最小触发。

**贡献。**
1. **一类新 bug**，带有干净的归因模型：可行但不安全的飞手输入（合法输入 ⇒ 飞控故障），以及一个区分真实 bug 与"主动指令坠毁"的非平凡性判据（内部、非饱和、*回中*）。
2. **一项刻画**：飞手输入鲁棒性地形——安全即平台、违例附近为幅值门控的通道各向异性——这解释了为何梯度/效率导向的方法在此无济于事。
3. **CADET**，一种控制通道导向的证伪 + 最小触发合成方法，以及一项**机制性结果**：通道导向是*必要的*——最小触发无法由通道无关搜索 + delta-debugging 恢复。
4. **真实 bug**，在 PX4 [及 ArduPilot] 中，以最小、可复现、回中的触发机动的形式交付。

## 2. 背景与问题模型  **[定义性]**

- **飞手输入与飞行模式。** 摇杆通道；受速率限制的连续信号；POSCTL/ALTCTL/LOITER 语义（哪个杆指令哪种运动）。
- **STL 契约与鲁棒度。** φ 作为 STL 属性；ρ 作为定量鲁棒度；回中后测量。
- **可行但不安全，形式化地。** `F`（可行/受速率限制）⊇ `P`（非自毁）与 `C`（飞控安全）。目标 `T = P \ C`。归因：输入 ∈ `P` ⇒ 归咎于飞控，而非飞手。自毁被排除（先例：PGFuzz）。
- **非平凡性。** 内部（离开摇杆限位）、非饱和，理想情况下回中。飞控缺陷的最强证据：飞行器在输入回中*之后*仍违例。

## 3. 动机性观察：鲁棒性地形  **[经验基础]**

本节给出为方法提供依据的刻画。*（所有 [已有] 均为单种子；[必做] 指需复现。）*

- **3.1 安全 = 平台。** 在良性输入周围，ρ 平坦，其实测梯度即为噪声（有效稀疏度 ≈ 25/40 = 纯噪声水平；噪声 ≈ 280× 名义步长）。梯度下降不下降。→ 推动结构无关的阶段 1。**[已有]**
- **3.2 违例附近 = 通道各向异性。** 用抗噪方向探针，边界法向集中在 roll+pitch（≈ 86% 的敏感度质量；通道参与比 ≈ 2.6/4），但在时间上稠密（窗口 PR ≈ 7.6/10）；活跃通道在不同条件下稳定，且与控制分配相符。→ 推动通道导向的阶段 3。**[已有]**
- **3.3 有 bug 价值的违例是薄目标。** 均匀随机找到大量违例但**没有一个是内部的**；它的违例是饱和/高幅值的（中位 max|θ| ≈ 0.83）。内部薄壳正是随机错过的部分。**[已有]**

*（诚实表述：我们报告通道集中性可能是幅值门控的，而非严格边界局部的；这不影响方法，方法在集中性出现的任何地方都加以利用。）*

## 4. CADET  **[方法 + 机制——论文的核心]**

- **4.1 概览。** 两段式结构：先以结构无关方式到达违例区域（因为平台），再利用通道各向异性来合成最小触发。
- **4.2 阶段 1 —— 结构无关到达。** 非梯度采样 / 契约延拓，得到一个鲁棒安全 / 鲁棒违例的跨界点对。2σ 的 J 次重复门限（边界为高噪声）。
- **4.3 阶段 2 —— 判据：属性→通道推导。** 从控制分配（哪些通道驱动受约束运动）推导活跃通道集 `A_φ`；用方向探针验证。*与 RouthSearch 的类比：* 一个由系统推导出的判据来结构化搜索。
- **4.4 阶段 3 —— 通道导向的内部边界搜索。** 搜索约简后的 通道×包络 空间；用符号包夹落到*内部*边界；违例天然最小。
- **4.5 阶段 4 —— 触发合成与刻画。** 约简到最温和的鲁棒、回中形式；输出触发机动 + 通道/时间"危险地图"。
- **4.6 通道导向为何必要（机制——务必突出）。**
  > 最小触发与稠密违例是输入空间中的*不同点*，二者不被任何保持鲁棒性的删除路径连接。Delta-debugging 只在 ρ 仍保持鲁棒违例时移除元胞；一个**温和**违例（ρ 接近 0）没有裕度，因此删除会越回安全侧，最小化器停滞在一个稠密输入上——而有 bug 价值的违例恰恰就是温和的那些。一个支撑很小的协同 roll/pitch 输入却可能是*深度*违例的；该点可通过**直接搜索通道约简空间**到达，而非通过最小化。这就是"先找任意违例，再最小化"失败、且通道导向为必要的原因。

本小节是全文的智识核心；§6 的 RQ2 是它的实验证明。

## 6. 评估  **[研究问题]**

开篇即声明：除非标注为已复现，否则下述结果均为**单种子/单场景探针**；多种子与 ArduPilot 在投稿前为 **[必做]**。

- **RQ1 —— CADET 能否找到随机错过的非平凡违例？**
  比较内部/回中违例产出：随机（0 个内部）vs. CADET（18 个内部；支撑 4–8；roll/pitch）。**[已有]**
- **RQ2 —— 它的触发是否最小，且通道导向是否*必要*？**（核心方法学结果）
  CADET 触发：支撑 4–8，两个通道。通道无关 + delta-debugging：仅 4/10 干净，中位支撑 14.5，半数情况下 throttle/yaw 残留，每个干净触发的代价约高一个数量级——*尤其在温和违例上失败*（§4.6 的机制）。**[已有；代价待按每个不同触发重新核算]**
- **RQ3 —— 鲁棒性地形。** 把 §3 作为结果呈现：平台 + 通道各向异性；说明鲁棒度下降（S-TaLiRo 风格）为何停滞。**[已有 + [必做] 复现]**
- **RQ4 —— 真实 bug。** 编目找到的最小触发，连同其通道/时间签名，在 PX4 [及 ArduPilot] 中；论证其影响与可复现性。**[PX4 已有；ArduPilot 为 [必做]]**
- **RQ5 —— 代价与产出。** 每个不同干净触发的代价 vs. 基线；不同触发族（trigger families）的产出。**[必做：按不同触发核算]**

**基线。** 均匀随机；鲁棒度引导的证伪（S-TaLiRo 风格）；通道无关 + delta-debugging；（可选）代理/覆盖引导的测试生成。**决定性指标：** 非平凡产出、触发干净度、可靠性、每个不同干净触发的代价——*而非*首次违例时间。

**诚实负面结果，作为定界报告（不掩埋）：**
- CADET 在找*任意*违例上**并不**更快（随机约 50 次查询）。**[诚实负面]**
- 它**不**产生比稀疏铺开搜索更低峰值幅值的输入。**[诚实负面]**
- 跨条件热启动**不**带来查询节省（边界本就便宜）。**[诚实负面]**
- 模式切换交接在此**不**是 bug 来源（PX4 POSCTL→LOITER 交接是干净的）。**[诚实负面]**

这些负面结果反而强化论文：它们表明贡献恰恰是*为一类新 bug 给出最小可解释触发*，而非一个过宽的效率主张。

## 7. 讨论与有效性威胁  **[诚实小节]**

- **单种子/单场景。** 首要威胁；由跨种子、跨场景的 **[必做]** 复现来缓解。
- **泛化性 / 平台。** 在 **[必做]** 的 ArduPilot 移植之前仅限 PX4；单一机架类。
- **通道推导。** 必须是原则性的（来自控制分配），而非逐属性拟合；由探针验证。若某属性的驱动通道含糊则有风险。
- **仿真到现实（sim-to-real）。** 违例在仿真中判定；其真实性继承了 sim-to-real 间隙。
- **理论。** 我们刻意*不*依赖可行性 / Nagumo（其 cell-sparse 预测在经验上已被证伪）；基础是实测的通道各向异性 + 控制分配。边界局部 vs. 幅值门控的问题尚开放，且不影响方法。
- **bug 的范畴。** 对触发做严重性/发生概率加权（真实飞手实际会做哪些机动）是未来工作。

## 8. 相关工作  **[定位]**

- **RouthSearch**（最近的谱系）：在 PID *参数*上做判据结构化搜索；CADET 是在*飞手输入*上做判据结构化搜索，以控制分配的通道结构作为判据。
- **RV fuzzing**（PGFuzz、RVFuzzer 等）：针对参数 / 指令 / 环境，报告策略/安全违例；CADET 针对*人工摇杆时序*，刻画边界，并合成*最小回中*触发。自毁排除沿用 PGFuzz。
- **CPS 证伪**（S-TaLiRo / 鲁棒度下降 + SA；代理引导、分类引导、学习引导）：共享鲁棒度 oracle 的表述；区别在于通道导向，以及*最小可解释触发合成*而非单一反例搜索。注意此处下降会在平台上停滞。
- **多样化 / 覆盖引导的反例**（WOGAN、决策树、覆盖探索器）与**风险/使用感知的证伪**：与"反例的价值"相关；我们的非平凡性 + 通道最小性判据是其领域特定的实例。

---

## 9. 结论

可行的飞手输入能用*简单、单通道的机动*破坏一个正确的飞控——包括在摇杆居中后仍持续的漂移。这些 bug 对随机搜索不可见，对先找后删不可恢复，因为最小触发无法通过从稠密违例中删除元胞而到达。CADET 通过沿驱动每个安全属性的控制通道导向搜索来找到它们，产出最小、人类可读的触发，以及一张"哪些机动危险"的刻画。产出这套方法的同一套纪律，也产出了一摞诚实负面结果（效率、热启动、切换 bug），它们使其主张更加锐利。
