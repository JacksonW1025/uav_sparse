# CADET：方法规范

> **CADET** = **C**ontrol-**A**llocation-**D**irected **E**xploration of **T**riggers（控制分配导向的触发探索）

**目标。** 给定一个固定增益 / 传感器 / 固件 / 混控器（mixer）/ 任务的飞控程序，以及一个用 STL 表达的飞行模式安全契约 φ，发现*可行但不安全的飞手输入*——即合法、受速率限制的人工摇杆序列，但会违反 φ——并将它们合成为最小化、人类可读、回中（return-to-neutral）的**触发机动（trigger maneuvers）**，同时给出违例边界的控制通道结构的刻画。

CADET 与 RouthSearch / MarginSearch 属于同一谱系的后继方法（原则性判据 + 结构化搜索 + 离线 oracle），其判据为**控制分配（通道→属性）结构**，而非稳定性或可行性（viability）条件。

---

## 1. 问题设定

- **输入。** 飞手输入 `U ∈ R^D`，采用固定参数化（此处 D = 40：10 个窗口 × 0.5 s × 4 个通道 {roll, pitch, yaw, throttle}，5 s 时域，随后接 8 s 回中尾段）。可行集 `F` 施加 `|u_t| ≤ L`（摇杆限位）以及逐步速率限制 `|u_t − u_{t−1}| ≤ R`——这就是*可行*（人类可发出）输入的定义。
- **Oracle。** 仿真器产生飞行日志 `Y(U)`；STL 监控器返回鲁棒度 `ρ_φ(Y(U))`（> 0 安全，< 0 违例）。所有属性都在*回中之后*（摇杆回到中位后）测量。
- **随机性。** `ρ` 含噪声；仿真器对每个输入重跑 J 次，汇总为 `(ρ_mean, ρ_std)`。
- **关注的几何结构。** `P ⊆ F` 是非自毁区域（不直接指令坠毁的输入）。`C` 是控制器安全集。目标为 `T = {U ∈ P : ρ_φ(U) < 0}`——即可行但不安全的输入。当一个触发位于 `P` 的内部（不在摇杆限位上）且理想情况下是*回中*的，它就是*非平凡的*。

---

## 2. 结构性前提（经实验确立，而非假设）

CADET 建立在先前实验确立的四个事实之上，**而不是**建立在可行性理论之上：

1. **安全即平台（Plateau-when-safe）。** 远离违例处，`ρ` 是平坦的，其梯度即为噪声。→ 梯度引导的搜索无法逃出安全区域；到达边界必须是结构无关（structure-agnostic）的。
2. **幅值门控、通道各向异性的敏感度。** 在违例附近，`ρ` 的敏感度集中在*少数控制通道*（即驱动该属性的那些通道——例如横向速度 ← roll/pitch），但在*时间上是稠密的*（一段连续的窗口带）。活跃通道在不同条件下保持稳定。
3. **非平凡违例是一个薄目标。** 随机搜索能找到违例，但它们是高幅值 / 饱和的；真正有 bug 价值的*内部*违例是随机搜索会错过的一层薄壳。
4. **最小触发不在删除路径上。** 一个干净、低支撑（low-support）、温和的违例，与一个稠密违例，是输入空间中的*不同点*，二者之间不存在保持鲁棒性的逐元删除路径。→ 从稠密违例出发的删除式最小化（delta-debugging）恰恰在温和违例上失败；干净的触发必须在通道约简空间中被**直接搜索**出来。

前提 2 与 4 正是搜索为何是*通道导向*而非*先找后删*的原因。

---

## 3. 算法

CADET 分五个阶段运行。阶段 0–1 以结构无关方式到达违例区域；阶段 2–4 利用通道结构来合成干净的触发。

### 阶段 0 —— 参数化与硬剪枝（先验）
将原始 tick 级输入约简为可行输入参数化（窗口 × 通道），并剪掉对目标模式可证明无效（inert）的通道-时间组（依据源码/文档）。这是*先验*的降维，而非搜索。

### 阶段 1 —— 结构无关地到达违例区域
由于安全区域是一个平台，使用**非梯度**移动来获得至少一个鲁棒违例点和一个鲁棒安全点，二者跨越边界：
- 对 `F` 进行均匀 / 协同的大幅值采样，和/或
- 契约延拓（先求解一个放松后的契约，再向真实契约收紧）。

输出：一个跨界点对 `(U_safe, U_viol)`，满足 `ρ_mean(U_safe) − 2σ > 0` 且 `ρ_mean(U_viol) + 2σ < 0`。

### 阶段 2 —— 推导属性相关的控制通道（判据）
从控制分配结构（哪些通道驱动该属性所约束的机体运动）出发，结合一个抗噪的方向探针（用大步长 δ 压过边界噪声；重复 J 次），辨识出承载边界法向敏感度的**活跃通道集** `A_φ ⊆ {channels}`（例如 `A_xy_velocity = {roll, pitch}`）。这是 RouthSearch 判据的对应物：先从系统推导出来，再经经验验证。

> 设计准则：从控制分配推导 `A_φ`，使方法*原则性且可迁移*，而非逐属性手工调参。用经验探针来*验证*该推导。

### 阶段 3 —— 通道导向的内部边界搜索
搜索这个**约简、结构化的空间**：仅在 `A_φ` 上有支撑的输入，按每个活跃通道的小*包络（envelope）*参数化（活跃通道平面内的方向、幅值、起始时刻、持续时长）。在此空间内：
- 使用**符号包夹（sign-bracketing）**（在一个鲁棒安全包络与一个鲁棒违例包络之间二分）落到**内部**边界——而非一个深度饱和的违例；
- 由于支撑*天然*很小，这里找到的违例已经是最小的（支撑 ≈ |A_φ| × 带宽）。

鲁棒性规则（强制）：仅当在 J 次重复下 `ρ_mean + 2σ < 0` 时，一个点才计为违例。边界附近 ρ-std 很大；该门限拒绝噪声造成的伪跨界，保持包夹诚实。

### 阶段 4 —— 触发合成与刻画
- **最小触发。** 在通道约简的结果内，做轻度的支撑/幅值约简，得到最温和的鲁棒形式；确认它是内部的、非饱和的，并（在可能时）是回中的——即违例是在摇杆已回到中位*之后*才被记录的。
- **边界刻画。** 记录活跃通道、时间带，以及每个属性越界时的幅值——形成一张"危险地图"，标明哪些简单机动是不安全的。

输出：一组最小、可解释、回中的触发机动（例如"在窗口 4–9 上做约 1 s 的半杆 pitch，然后松杆 → 回中后 xy 速度超过 1.0 m/s"），每个都带有其鲁棒 ρ 以及其通道/时间签名，外加边界刻画。

---

## 4. 每个阶段为何必要（与证据的对应）

| 阶段 | 它回应的前提 | 缺了它会出什么问题 |
|---|---|---|
| 1（结构无关到达） | 安全即平台 | 梯度下降在平台上停滞（SPH/persistence 失败） |
| 2（通道推导） | 通道各向异性 | 在全部 40 维上搜索会把预算浪费在无效通道上；触发仍然稠密 |
| 3（内部包夹） | 薄目标前提 | 随机只找到平凡/饱和的违例，找不到有 bug 价值的内部违例 |
| 4（直接合成，而非最小化） | 最小触发不在删除路径上 | 先找后删（ddmin）不可靠，且在温和违例上代价约高 10× |

---

## 5. 参数与停止条件

- `J`（每个 ρ 的重复次数）：5（边界为高噪声）。在 2σ 处设门限。
- 方向探针步长 `δ`：大（≈ 0.2）以用于通道辨识（SNR ∝ δ；δ 太小则在边界处被噪声主导）。
- 摇杆限位 `L`、速率限制 `R`：固定为可行输入定义（公开披露；非调参得来）。
- 阶段 3 在内部边界被包夹到由 ρ-噪声主导的容差时停止；阶段 4 在任何进一步的支撑/幅值约简都无法保持鲁棒违例时停止。
- **预算单位：** 以 J=5 个点计数。报告**每个不同（distinct）触发**的代价（按不同的活跃通道/时间/包络签名区分），而非每次原始评估的代价。

---

## 6. CADET 刻意**不**主张的内容

- 不主张用更少的查询去找到*任意*违例（随机本就便宜）。
- 不主张产生比稀疏铺开的通道无关搜索更低峰值幅值的输入。
- 不主张可行性 / Nagumo 保证（cell-sparse 的预测已被证伪；基础是经验上的通道各向异性 + 控制分配）。
- 不是模式切换交接（transition-handoff）的 bug 检测器（测试中交接是干净的）。
- 在被复现之前，仅为单种子/单场景。

它的主张是精确且有支撑的：**对一类全新 bug，可靠、低代价地合成最小、可解释、回中的触发**——这是随机搜索、以及通道无关的证伪 + 最小化都无法产生的。

---

## 7. 评估中必须对比的基线

- **均匀随机**（它能否找到内部/非平凡违例？——它不应该能）。
- **鲁棒度引导的证伪**（S-TaLiRo 风格的下降 + 模拟退火；在平台上停滞）。
- **通道无关搜索 + delta-debugging**（必要性基线：先找后删能否恢复出干净触发？——不可靠且昂贵，在温和违例上尤甚）。
-（可选）代理模型 / 分类引导的证伪；覆盖/多样性测试生成器。

决定性的比较项是*非平凡违例产出率*、*触发的干净度*（支撑、活跃通道）、*可靠性*，以及*每个不同干净触发的代价*——**而非**首次违例时间（time-to-first-violation）。

---

## 8. 任何论文主张之前必须完成的验证

1. **复现**阶段 1–4 及所有对比，跨 ≥ 2 个额外种子。
2. **泛化**到第二个属性（`xy_drift`）以及 **ArduPilot**（跨平台 bug 类）。
3. **原则化**阶段 2 的通道推导（使其来自控制分配，从而是一种方法，而非逐属性调参），并在每个新属性上用经验探针验证它。
4. **核算**每个不同触发的代价（取代单种子下摊销得到的比值）。
