## 2. 从未改动的实验地基

- 场景：PX4 `px4_position`（POSCTL）。
- 输入参数化：`U ∈ R^40` = 10 窗 × 0.5 s × 4 通道 {roll, pitch, yaw, throttle}，5 s 主动段，随后 8 s 回中尾段（`t_neutral = 5.0 s`，尾段 [5,13] s）。
- 可行集 `F`：限位 `|u_t| ≤ 1.0` + 逐步速率限制 `|u_t − u_{t−1}| ≤ R`（= 人类可发出）。
- Oracle：仿真器产飞行日志，STL 监控器返回鲁棒度 ρ（>0 安全，<0 违例），含噪声。
- 重复：J = 5。鲁棒 2σ 门：违例 `ρ_mean + 2σ < 0`，安全 `ρ_mean − 2σ > 0`，否则 noise_band。
- 分类：`INTERIOR_MAX_ABS = 0.5`，`SATURATED_MIN_ABS = 0.9`，`support_abs_threshold = 0.1`。
- 每臂 80 个 J=5 点；方向探针 `δ_probe = 0.2`。

**病根（v4 确认）**：原始 `post_neutral_xy_velocity` 的鲁棒度，代码 `compute_robustness` 取的是**整条尾段 [5,13] 的峰值 |v_xy|**，阈值 1.0 m/s。整段峰值永远落在 [5,7]（松杆瞬间，≈ 机动巡航速度），故任何巡航过 1 m/s 的机动都会被判违例——这就是把 945 个点全部误判成"违例"的窗口混叠（§3 第十至十二幕坐实）。

代码层变化（历史）：包 `src/sparsepilot/ → src/cadet/`；写死的 `CHANNEL_RELEVANT_SET` 已替换为 `derive_A_phi(property)`（xy_velocity→{roll,pitch}、climb_rate→{throttle}、yaw_rate→{yaw}）。

## 3. 完整时间线（第一幕 ~ 第十二幕）

项目的智识价值在于：方法不是设计出来的，是从一连串被证伪的假设里幸存下来的——而最后两幕显示，连"幸存者"（锚点违例）也站在一个度量假象上，并被自己的复核纪律亲手推翻。

### 第一幕 —— SPH（支撑持续性假设）：发现"安全即平台"
- 假设：违例附近的支撑结构在扰动下持续。**证伪。**
- 真发现：远离违例处，ρ 是**平台**——回中输入周围 top-4 梯度质量仅 0.23–0.30，有效稀疏度 ≈ 25（恰为 40 维纯噪声值），梯度几乎全是噪声。
- 推论：梯度搜索在平台停滞 ⇒ 到达边界必须**结构无关** ⇒ 两段式架构。
- 来源：`fd_snapshot` / `repeated_fd` / `persistence_pilot`。

### 第二幕 —— MarginSearch（H1/H2/H3）
- H1 cell-sparse 可行性（Nagumo 式）：**死**。
- 幸存资产：**通道各向异性**（边界法向在通道轴稀疏，roll+pitch ≈ 86% 方向敏感度质量、时间轴稠密）。
- H2 热启动摊销：**死**（加速 1.01–1.65×）；带出"违例很便宜就能找到"。
- H3 模式切换交接 bug：**死**（0 个鲁棒切换违例）。
- 来源：`margin_stage1_redo`(δ=0.2) / `route1_h2_campaign` / `h3_transition`。

### 第三幕 —— 方向 A（seed 0），命名定为 CADET
- 三臂探针：随机内部违例 **0**、通道无关 **7**、通道导向 **18** → 当时确立"薄目标"与"触发干净度"。
- ddmin 必要性：seed 0 上 ddmin 仅恢复 4/10 起点的干净触发，支撑中位 14.5，约 10× 代价 → *当时*判"通道导向必要"。
- 命名纠偏：ChannelSearch → **CADET**。
- 来源：`direction_a_probe` / `direction_a_ddmin`。
- ⚠️ **v4 确认**：本幕"违例"全部用整段取峰度量判定，经第十二幕连锁核对，在终端窗下**全部 safe**。本幕据以成立的薄目标/必要性结论失效。

### 第四幕 —— 跨种子复现（seed 0/1/2）：必要性与数量占优倒下
跨 3/3 种子*当时*幸存：薄目标（Arm A 内部违例 0/0/0）、通道方向性（Arm C 内部违例 ⊆ {roll,pitch}）、通道无关稠密（Arm B 支撑中位 27/25.5/27）。
跨种子倒下：**必要性已死**（distinct 干净触发 CADET 7/0/1 vs ddmin 4/2/4）；**support 4–8 稀疏已死**（中位 6/18/7.5）；**最温和幅值非常数**（CV 22.7%）。
recut：原始 Arm-C 计数被幅值二分灌水；按 distinct 签名后 CADET 7/4/3 vs ddmin 5/7/8；每 distinct 代价 CADET 3/3 更便宜（ddmin/CADET = 8.28/3.22/2.07×，退化）。裁决：脊柱降为"代价优势"，"necessary" 删除。
时长–幅值权衡：Spearman ρ ≈ −0.32 ~ −0.37，"温和"与"稀疏"冲突。
来源：`direction_a_*_seed{0,1,2}_v0`、`seed_replication_*`、`recut_*`。
- ⚠️ **v4 确认**：这些"违例"同样基于整段取峰；经第十二幕，三臂在 3 seed 下终端窗 survivor 均为 0。"代价优势"的比较对象本身不存在，故 moot。

### 第五幕 —— bug 类重定义：回中后残余运动 + 契约 provenance
- 收敛到单一 bug 类：**回中后残余运动**（杆回中后控制器未能把对应机体速率驱动到零），按通道实例化。位移类度量（alt_drift/xy_drift）排除。
- 契约 provenance：PX4 文档明确——杆回中应**主动刹停、回平、锁定到三维位置**；满杆以 MPC_ACC_HOR_MAX 加速、衰减到 MPC_VEL_MANUAL。先例 PGFuzz。
- 阈值绑 PX4 参数（0.10 分数）：xy 1.0、climb 0.3、yaw 0.2618；MPC_HOLD_DZ=0.1。
- 两层 severity：Tier 1 = 终端窗 [11,13]s 残余越阈；Tier 2 = Tier 1 且尾段不收敛。
- ⚠️ **v4 注**：终端窗 [11,13] 在这里第一次被写下为"正确"窗口，但既有 oracle 仍用整段 [5,13] 取峰——这处口径不一致正是第十至十二幕的全部根源。**讽刺的是，第五幕自己定义的正确窗口，最终推翻了项目的锚点。**

### 第六幕 —— 通道迁移检验：判据身份会迁移，bug 类不迁移
- alt_drift（throttle）：饱和 +throttle 深度违例 alt（ρ=−1.363），−throttle 安全。H-alt-1 判据迁移**确认**（throttle 91%）；H-alt-2 内部合成**证伪**（10 鲁棒违例但 0 内部）。alt 是位置度量 → 操作化错误。
- 残余速率 climb/yaw：Phase 0 双双停止，连饱和都不可 Tier-1 违例。climb 终端残余 ≈ 0.026 m/s（阈值 0.3）；yaw ≈ 0.015–0.017 rad/s（阈值 0.262）；对阈值鲁棒。机制：竖直靠直接推力、偏航靠直接力矩，及时刹停。
- 裁决：bug 多通道迁移**已死**；判据通道身份迁移，bug 不迁移。
- 来源：`alt_drift_*`、`residual_rate_*`。
- ⚠️ **v4 注**：此幕用的就是终端窗 [11,13]，所以"climb/yaw 终端残余远低于阈值"可信——它其实已经预示了：水平在终端窗下也刹得很好（饱和 ~0.31、内部 ~0.04，全 < 1.0）。当时没有把这个信号回推到锚点上，是事后看最该早一步的地方。

### 第七幕 —— 跨通道耦合 / RGA：判据升级的最后一次原则性尝试
- 动机：对角判据显而易见、新意薄；想用 RGA 非对角/交互预测非显然的跨轴脆弱性。
- RGA：`Λ = G∘(G⁻¹)ᵀ`，本项目用经验类比（非线性、回中残余）。
- Step A（exploratory，预注册 provenance 有缺口）：G 只有 vx/vy 两行活；真信号 throttle→vx=0.041（roll→vx 0.0985 的 41%）。但 Λ 条件数 98.65、近秩 2，负值/交互候选**大概率数值伪影**。Step B 未跑。
- 来源：`cross_coupling_summary/`。
- ⚠️ **v4 确认**：其指向的"水平速度刹不住"已被第十二幕证伪，故耦合候选连带失去意义。

### 第八幕 —— 契约–输入–bug 的拓宽（概念转向）
- **PGFuzz 观察坐实**：PGFuzz 契约是覆盖指令/参数/状态的 MTL 策略（广度型），不是残余速度形式；我们的"残余速度"只是其中一格，对"刹停"不算太严而是**太窄**。
- **文档契约集**（PX4 位置模式）：回中→刹停 + 回平 + 锁位（一句话捆 3 个，只做了刹停）；机动中→速度上限（MPC_VEL_MANUAL/MPC_XY_VEL_MAX）、倾角（MPC_MAN_TILT_MAX=35°）、加速度（MPC_ACC_HOR_MAX）。
- **逻辑关系**：契约定义 oracle、输入定义可行空间、bug = (输入形态 × 契约) 网格的格子；能评估哪个契约取决于输入形态。
- **输入分类**：I1 脉冲回中、I2 持续保持、I3 多轴协同、(I4 反向/doublet)。

### 第九幕 —— 契约网格 Phase-0（G01–G12，seed 0，exploratory）
- 8 格饱和处 robust_safe（C1 刹停全部、C3 包络 roll/pitch、C4 耦合全部）；2 格 noise_band（C2 回平残余倾角 ~3.3° vs 3.5°；C3 throttle 上行爬升 3.03 vs 3.0）；0 格饱和处 robust_violation。
- **关键方法学要点**：饱和探针"必要不充分"，看不到内部违例。G01/G02 饱和 roll/pitch 残余水平速度仅 0.31 m/s（安全），与当时相信的"内部 >1" 矛盾——这一矛盾直接触发第十幕核对。

## 4. 契约–输入–bug 三元组（可复用的思考框架）

换不换方向都值得保留：
- **契约**：什么叫"对"，从被测系统官方文档导出（PGFuzz 式），定义 oracle。**纪律**：阈值绑可查参数、看数据前冻结、**操作化窗口要对**（本项目最大教训：窗口选择能凭空造出/抹掉违例）。
- **输入**：什么叫"合法可发出"，受限位 + 速率约束的飞手机动，定义搜索空间。分类：脉冲回中 / 持续保持 / 多轴协同 / 反向。
- **bug** = ∃ 合法输入违反某契约 = (输入形态 × 契约) 网格的格子。
- **三者不可分**：能评估哪个契约取决于输入形态；控制分配判据结构化每格的通道搜索。
- **PX4 位置模式文档契约集**（可复用清单）：回中→刹停/回平/锁位；机动→速度上限/倾角/加速度；另有死区（MPC_HOLD_DZ）、近地限速（MPC_LAND_*）。

## 6. 方法现状 + 对优秀 ISSTA 论文的诚实差距

**现状一句话**：方法不指向一个 bug，而指向一个（已确认的）"PX4 位置模式对可行飞手输入在这组契约/种子下稳健"的负面结论 + 刹车动力学刻画 + 一套自查方法论。

**ISSTA 标杆对照**（"找 bug"论文需要：真实、可复现、最好开发者确认的 bug；对 SOTA 明确推进；多目标/配置泛化）：

| 维度 | 状态 |
|---|---|
| 问题意义（飞手输入安全） | 够：选题好 |
| 方法新意（控制分配导向、可行输入证伪） | 概念有，但宣称优势已被自身实验打掉 |
| 复现 / 严谨性（预注册、复核） | 够，是亮点 |
| 与基线比较（随机、ddmin） | 做了，但无真 bug 则比较 moot |
| **真实、被确认的 bug** | **没有——锚点已确认是假象。致命缺口** |
| 泛化（多平台/模式/种子） | 不够：单平台、单模式；ArduPilot 零证据 |

**结论**：差距不是"补种子、润色"的化妆问题，而是**最关键那格是空的**：没有任何经得起复核的违例（已确认）。这批数据里既不成"找 bug 论文"，也不成"方法论文"。够的只有外围；缺的是核心。

## 9. 诚实威胁 + 方法论教训

**威胁**：中心对象已证否（本阶段定论）；单平台/单模式；ArduPilot 零证据；sim-to-real；判据虽原则化但 RGA 升级不可信（cond 98.65）。

**可迁移的方法论教训**（这是本阶段最稳的产出，带走）：
1. **尾段窗口混叠（核心教训）**：STL 残余/刹停类契约，若评估窗口从"指令撤除瞬间"起算并整段取峰，会把正常的主动收敛暂态误判为违例。本项目里它制造了 945 个跨 3 seed 稳定复现、扛过多轮审视的"鲁棒违例"，全部是假象。**正确做法是终端窗（或显式分离暂态与稳态残余），且看数据前固定。**
2. **"鲁棒 + 跨种子稳定复现"不等于"真"**：一个假象只要来自度量本身，就会稳定、可复现、跨种子——稳定性不是真实性的证据。
3. **饱和探针必要不充分**：饱和安全 ≠ 无 bug；反过来（本项目）内部也不违例时，要警惕是度量假象而非"薄目标"。
4. **预注册先提交再跑**，否则只能 exploratory。
5. **复核能救命**：单调窗口论证 + "拿已知违例 θ 过另一套忠实度量重判" + 连锁核对，纯后处理 29 s 就在投稿前推翻了假象。这套纪律本身是项目最有价值的产出。
