> 本文是 `CADET_工作全景梳理`（v1/v2）的继任版，目标是把从 SPH 开始到此刻的**全部**工作过程、结论与证据合并成一份可独立阅读的档案：既供作者本人理清思路，也供迁移到新对话作上下文。
>
> **v3 与 v2 的根本区别**：v2 收尾时，项目的"锚点"——一个可行飞手输入触发的契约违例（回中后残余水平速度 >1 m/s）——被认为是跨 3 个种子站得住的硬结果，方法论文的整条脊柱（薄目标、通道导向、每触发代价）都建立在它之上。**v3 记录了一次口径核对，它强烈指向：这个锚点违例很可能是一个度量窗口假象**（整段尾段取峰抓到了松杆后的正常刹车暂态，而飞行器其实在几秒内刹停了）。这把问题从"方法主张站不站得住"升级到"**bug 到底存不存在**"。

> **写作纪律**（沿用并强化）：每个定量数字标注来源（seed、runner、artifact）；每个被证伪的主张明确标"已死"；每个"我们曾希望成立但数据不支持"的主张单独标记；这一版新增——**凡是依赖锚点违例为真的结论，一律标为"待连锁核对"**。

## 1. 一句话现状（v3）与谱系定位

**v1 一句话**（最乐观）："CADET 找出最温和的合法摇杆机动，用最少的控制通道表达，能在飞手回中后仍破坏一个飞行模式安全契约——随机错过，通道无关 + ddmin 无法可靠恢复。"

**v2 一句话**（数据校准后，但仍信锚点）："针对一个文档契约背书的新 bug（PX4 位置模式水平速度回中后刹不住），用控制分配判据把 40 维搜索导向 roll/pitch，可靠合成干净、回中的触发；每触发比 ddmin 便宜 2–8×，随机够不到。"

**v3 一句话**（锚点被质疑后）：

> CADET 是一套**原则性的、有纪律的飞手输入证伪框架**（控制分配导向 + 文档契约 + 可行输入分类 + 预注册 + 复核）。把它严格应用到 PX4 位置模式后，**此前认定的锚点违例经口径核对极可能是度量假象**；目前真正站得住的是：一套刹车动力学/稳健性地形的**刻画**，一个关于 STL 证伪如何被尾段窗口混叠误导的**方法论教训**，以及一个（待确认的）"该模式对可行输入稳健"的**负面结论**。

**谱系（不变）。** RouthSearch（ISSTA 2025）同门后继：原则性判据 + 结构化搜索 + 离线鲁棒度 oracle。RouthSearch 用 Routh–Hurwitz 结构化对 PID *参数*的搜索，找到 PID 误配 + 3 个 PX4/ArduPilot bug；CADET 固定一切、搜索*飞手输入*，判据换成**控制分配（通道→属性）结构**。PGFuzz（NDSS 2021）是更远的近邻：从官方文档导出 MTL 策略、偏离即 logic bug，48 小时跨 PX4/ArduPilot/Paparazzi 找到 156 个 bug——它的力量来自**契约的广度**，与我们的"窄而深"恰成对照。

## 2. 从未改动的实验地基

整条线上这些设置始终冻结：
- 场景：PX4 `px4_position`（POSCTL）。
- 输入参数化：`U ∈ R^40` = 10 窗 × 0.5 s × 4 通道 {roll, pitch, yaw, throttle}，5 s 主动段，随后 8 s 回中尾段（`t_neutral = 5.0 s`，尾段 [5,13] s）。
- 可行集 `F`：限位 `|u_t| ≤ 1.0` + 逐步速率限制 `|u_t − u_{t−1}| ≤ R`（= 人类可发出的定义）。
- Oracle：仿真器产飞行日志，STL 监控器返回鲁棒度 ρ（>0 安全，<0 违例），含噪声。
- 重复：J = 5。鲁棒 2σ 门：违例 `ρ_mean + 2σ < 0`，安全 `ρ_mean − 2σ > 0`，否则 noise_band。
- 分类：`INTERIOR_MAX_ABS = 0.5`，`SATURATED_MIN_ABS = 0.9`，`support_abs_threshold = 0.1`。
- 每臂 80 个 J=5 点；方向探针 `δ_probe = 0.2`。

**注意（v3 关键）**：原始 `post_neutral_xy_velocity` 的鲁棒度，代码里 `compute_robustness` 取的是**整条尾段 [5,13] 的峰值 |v_xy|**，阈值 1.0 m/s。**正是这个"整段取峰"的窗口选择，在 §3 第十一幕被发现会把松杆后的刹车暂态误判为违例。** 这一点是 v3 全部重估的技术根源。

代码层变化（历史）：包 `src/sparsepilot/ → src/cadet/` 改名已落地；写死的 `CHANNEL_RELEVANT_SET` 已替换为 `derive_A_phi(property)`（xy_velocity→{roll,pitch}、climb_rate→{throttle}、yaw_rate→{yaw}）。

## 3. 完整时间线（第一幕 ~ 第十一幕）

项目的智识价值在于：方法不是设计出来的，是从一连串被证伪的假设里幸存下来的——而最后一幕显示，连"幸存者"也可能站在一个度量假象上。

### 第一幕 —— SPH（支撑持续性假设）：发现"安全即平台"
- 假设：违例附近的支撑结构在扰动下持续。**证伪。**
- 真发现：远离违例处，ρ 是**平台**——回中输入周围 top-4 梯度质量仅 0.23–0.30，有效稀疏度 ≈ 25（恰为 40 维纯噪声值），梯度几乎全是噪声。
- 推论：梯度引导搜索在平台上停滞 ⇒ 到达违例边界必须**结构无关** ⇒ 两段式架构（先结构无关到达、再结构化合成）。
- 来源：`fd_snapshot` / `repeated_fd` / `persistence_pilot`。

### 第二幕 —— MarginSearch（H1/H2/H3）
- H1 cell-sparse 可行性（Nagumo 式）形式：**死**。
- 幸存资产：**通道各向异性**——违例边界法向在通道轴上稀疏（roll+pitch ≈ 86% 方向敏感度质量）、时间轴上稠密，活跃通道跨条件稳定。
- H2 热启动摊销：**死**（加速仅 1.01–1.65×）；但带出关键观察"**违例很便宜就能找到**"。
- H3 模式切换交接 bug：**死**（0 个鲁棒切换违例；交接是干净的）。
- 来源：`margin_stage1_redo`(δ=0.2) / `route1_h2_campaign` / `h3_transition`。

### 第三幕 —— 方向 A（seed 0），命名定为 CADET
- 三臂探针：随机内部违例 **0**、通道无关 **7**、通道导向 **18** → 当时确立"薄目标"与"触发干净度"。
- ddmin 必要性：seed 0 上 ddmin 仅恢复 4/10 起点的干净触发，支撑中位 14.5，约 10× 代价 → *当时*判"通道导向必要"。
- 命名纠偏：ChannelSearch → **CADET**（判据是控制分配，不是"通道"这个维度）。
- 来源：`direction_a_probe` / `direction_a_ddmin`。
- ⚠️ v3 注：本幕的"违例"全部用整段取峰度量判定，故其结论受第十一幕影响（见下）。

### 第四幕 —— 跨种子复现（seed 0/1/2）：必要性与数量占优双双倒下
跨 3/3 种子**幸存**：薄目标（Arm A 内部违例 0/0/0）、通道方向性（Arm C 内部违例 ⊆ {roll,pitch}）、通道无关必然稠密（Arm B 支撑中位 27/25.5/27）。
跨种子**倒下**：
- **"必要性"已死**：distinct 干净触发 CADET 7/0/1 vs ddmin 4/2/4——ddmin 在 2/3 种子 ≥ CADET。
- **"support 4–8 稀疏"已死**：仅 seed 0 成立；三种子中位 6/18/7.5。
- **"最温和幅值是常数"不成立**：CV 22.7%，range 0.28–0.45。
自由重切（recut）：原始 Arm-C 计数被幅值二分重复灌水；按 distinct 签名（通道集+窗口带+符号，排幅值）后 CADET 7/4/3 vs ddmin 5/7/8（CADET 仅 seed0 多）；每 distinct 触发代价 CADET 3/3 更便宜（ddmin/CADET = 8.28/3.22/2.07×，倍数退化）。
裁决：脊柱降级为"**代价优势，非数量、非必要**"，"necessary" 全文删除。
时长–幅值权衡：Spearman ρ ≈ −0.32 ~ −0.37，"温和"与"稀疏"相互冲突；support 不稳定大半是随机时长采样的搜索伪影。
来源：`direction_a_*_seed{0,1,2}_v0`、`seed_replication_*`、`recut_channel_gentle_v0`、`recut_distinct_v0`。
⚠️ v3 注：这些"违例"同样基于整段取峰度量。

### 第五幕 —— bug 类重定义：回中后残余运动 + 契约 provenance
- 从"违反任意 φ"收敛到机制同质的单一 bug 类：**回中后残余运动**——飞手把杆回中后，控制器未能把对应机体*速率*驱动到零。一个 bug 类，按通道实例化。
- 位移类度量（alt_drift/xy_drift）排除：位置合法保持是正确行为。
- 契约 provenance（曾被视为最强一块）：PX4 位置模式官方文档明确——杆回中时飞行器应**主动刹停、回平、并锁定到三维位置**，对抗风等外力；满杆以 MPC_ACC_HOR_MAX 加速、衰减到 MPC_VEL_MANUAL。先例 PGFuzz：从官方文档导出策略、偏离即 logic bug。
- 阈值绑定 PX4 参数（0.10 固定分数）：xy 1.0 m/s（MPC_VEL_MANUAL=10）、climb 0.3 m/s（MPC_Z_VEL_MAX_UP=3.0）、yaw 0.2618 rad/s（MPC_MAN_Y_MAX=2.618）；MPC_HOLD_DZ=0.1。
- 两层 severity：Tier 1 = 终端窗 [11,13]s 残余越阈；Tier 2 = Tier 1 且尾段 [5,13]s 不收敛。
- ⚠️ v3 注：终端窗 [11,13] 是这里**第一次**被明确写下的"正确"窗口；但 §2 的既有 oracle 仍用整段 [5,13] 取峰——两者口径不一致正是第十一幕的爆点。

### 第六幕 —— 通道迁移检验：判据身份会迁移，bug 类不迁移
- **alt_drift（throttle）**：饱和 +throttle 深度违例 alt（ρ=−1.363），−throttle 安全（非对称）。H-alt-1 判据迁移**确认**（throttle 91% 边界法向质量）；H-alt-2 内部合成**证伪**（10 个鲁棒违例但 0 个内部，全饱和）。alt 是位置度量 → 操作化错误。
- **残余速率 climb/yaw**：Phase 0 双双停止，连饱和都不可 Tier-1 违例。climb 终端残余 ≈ 0.026 m/s（阈值 0.3，约 10× 之下）；yaw ≈ 0.015–0.017 rad/s（阈值 0.262，约 17× 之下）；对阈值选择鲁棒。机制：竖直靠直接推力、偏航靠直接力矩，及时刹停；唯水平靠回倾、带动量，刹得慢。
- 裁决：**bug 多通道迁移已死**；判据的通道身份会迁移，bug 本身不迁移；bug 是水平速度独有。
- 来源：`alt_drift_*`、`residual_rate_prereg.md`、`residual_rate_summary/`。
- ⚠️ v3 注：此幕用的是终端窗 [11,13]——所以"climb/yaw 终端残余远低于阈值"是可信的；它也间接预示了水平在终端窗其实也刹得不错（见第十/十一幕）。

### 第七幕 —— 跨通道耦合 / RGA：判据升级的最后一次原则性尝试
- 动机：对角判据（直接通道）显而易见，方法新意薄；想用 RGA 非对角/交互结构预测"非显然"的跨轴脆弱性。
- RGA：`Λ = G∘(G⁻¹)ᵀ`，对角=直接配对、非对角=耦合、负值=危险配对。本项目用**经验 RGA 类比**（非线性、回中残余），非紧定理。
- Step A（exploratory，预注册 provenance 有缺口）：G 矩阵只有 vx/vy 两行活；真信号 **throttle→vx = 0.041（roll→vx 0.0985 的约 41%）、throttle→vy = 0.026** → 一个真实 throttle→水平耦合候选。但 RGA Λ **条件数 98.65、近秩 2**，负值/交互候选（C002/C003）**大概率是数值伪影**。Step B 未跑。
- 现实天花板：即便确认，仍是同一个"水平速度刹不住"经非显然 throttle 路径触发——适度加厚，非新 bug、非多通道迁移。
- 来源：`cross_coupling_summary/`。
- ⚠️ v3 注：这里的"水平速度刹不住"本身在第十一幕被质疑——所以此幕的"耦合候选"也连带失去意义。

### 第八幕 —— 契约–输入–bug 的拓宽（概念转向）
作者提出"小而美"缺厚度，三件事（契约/输入/bug）密不可分，应适度拓宽。查文档后确认：
- **PGFuzz 观察坐实**：PGFuzz 契约是覆盖指令/参数/状态的 MTL 策略（广度型，如"温度>100°C 必触发 failsafe"、降落伞最低高度），**不是**残余速度形式。我们的"残余速度"只是其中一格，且对"刹停"而言不算太严、而是**太窄**。
- **文档契约集**（PX4 位置模式，全部官方文档导出）：回中 → 刹停（速度→0）+ 回平（姿态→水平）+ 锁位（保持 x,y,z）——一句话捆了 3 个契约，我们只做了刹停；机动中 → 速度上限（MPC_VEL_MANUAL / 硬限 MPC_XY_VEL_MAX）、倾角包络（MPC_MAN_TILT_MAX=35°）、加速度（MPC_ACC_HOR_MAX）。
- **逻辑关系**：契约定义 oracle、输入定义可行空间、bug = (输入形态 × 契约) 网格里的一个格子；你能*评估*哪个契约取决于输入形态（刹停契约需回中输入；速度上限需持续打杆；轴独立需多轴协同）。控制分配判据告诉你每格搜哪些通道。
- **输入分类**：I1 脉冲回中、I2 持续保持、I3 多轴协同、(I4 反向/doublet)。

### 第九幕 —— 契约网格 Phase-0（G01–G12，seed 0，exploratory）
对 4 个契约 × 3 类输入的 12 个格子做便宜的饱和探针筛查：
- **8 格饱和处 robust_safe**：C1 刹停（roll/pitch/throttle/yaw 全部）、C3 包络（roll/pitch 速度 6.5/12、倾角 0.43/0.61，远在限内）、C4 耦合（throttle 单独 0.027、协同最多 0.35，都安全）。
- **2 格 noise_band**：C2 回平（饱和 roll/pitch 残余倾角 ~0.057 rad ≈ 3.3°，阈值 0.061 rad = 3.5°，部分重复越阈）；C3 throttle 上行（爬升 3.03 m/s vs 上限 3.0，rho 均值已为负、仅因 2σ 跨零记 noise）。
- **0 格饱和处 robust_violation。**
- **关键方法学要点**：饱和探针是"必要不充分"筛子，**看不到内部违例**。证据：G01/G02 饱和 roll/pitch 残余水平速度仅 0.31 m/s（安全），而我们当时相信内部 >1 m/s。故 `phase1_eligible` 全判 False 对 C1 是**漏判**。这一矛盾直接触发了第十幕的核对。
- 参数：MPC_XY_VEL_MAX=12、MPC_MAN_TILT_MAX=0.611 rad。
- 来源：`contract_grid_summary/`。

### 第十幕 —— 非单调性 / 口径核对：锚点 bug 很可能是度量假象（决定性）
为厘清"饱和 0.31 安全"与"内部 >1 违例"的矛盾，取 5 个已知内部违例 θ（覆盖 roll、pitch、roll+pitch、双符号）+ 2 个饱和对照，同批重仿真，用新旧两套度量并排，并报尾段四个子窗的衰减剖面。

**衰减剖面（峰值 |v_xy|，m/s）：**

| θ | [5,7] | [7,9] | [9,11] | [11,13] |
|---|---:|---:|---:|---:|
| eval164 (pitch,roll) | 0.876 | 0.080 | 0.040 | 0.044 |
| eval174 (pitch) | 0.925 | 0.073 | 0.047 | 0.050 |
| eval182 (pitch,roll) | 1.205 | 0.209 | 0.048 | 0.044 |
| eval199 (pitch,roll) | 1.120 | 0.238 | 0.052 | 0.045 |
| eval234 (roll) | 0.963 | 0.076 | 0.048 | 0.053 |
| 饱和 G01 (roll) | 7.150 | 3.924 | 0.362 | 0.305 |
| 饱和 G02 (pitch) | 7.222 | 3.682 | 0.274 | 0.323 |

**核对结论：**
1. **飞行器会刹停。** 速度峰值全在 [5,7]（松杆后头两秒），两秒内掉到 ~0.08，到 [9,13] 基本归零（~0.04）。连饱和从 7 m/s 刹到终端 0.31。
2. **原始度量测的是松杆瞬态。** 算术核对：原始 rho = 1.0 − [5,7] 峰值（整段峰值永远落在 [5,7]）。[5,7] 峰值 ≈ 机动在松杆瞬间的巡航速度。所以"整段取峰"会把*任何巡航速度超过 1 m/s 的机动*判成违例——这是窗口混叠，不是"刹不住"。
3. **在终端窗 [11,13] 度量下，5 个已知违例无一违例**（内部 0.04、饱和 0.31，都远低于 1.0）。
4. **噪声脆弱**：fresh 重跑系统性比存档更不违例（rho +0.3 ~ +0.5），原始度量下 5 个里只剩 1 个（eval182）还能鲁棒违例。
5. **为什么不能改用 [5,7] 窗口救回**：那是正常主动刹车暂态，文档承诺的是 active（非瞬时）刹停；挪窗口保 bug 等于把正常减速判成故障，且是看了数据再调窗口——违反预注册纪律。

**裁决**：口径不一致；锚点"残余水平速度违例"**极可能是整段取峰度量制造的假象 + 噪声**。忠实的终端残余度量下，无违例。
来源：`monotonicity_check_summary/`。

### 第十一幕 —— 第三次清算：中心对象的存在性被质疑
- 此前每一轮清算埋的是结构/效率/迁移/方法主张；这一轮埋的可能是**中心对象本身**——"一个可行飞手输入触发的契约违例"。
- **连锁影响**：薄目标（随机错过内部违例）和通道导向"代价优势"很可能一起继承假象——Arm C 专门构造持续巡航包络（→ 触发松杆瞬态混叠），随机很少构造，所以"随机找不到内部违例"未必是有意义的薄目标，而是"只有持续包络才触发那个混叠"。
- **尚未运行的确认**：连锁核对（终端窗重判三臂 + ddmin；seed 1/2 衰减抽查）。跑完才知道"摆正窗口后还剩多少"。
- 这是导师层面的决策点。

## 4. 契约–输入–bug 三元组（可复用的思考框架）

无论后续是否换方向，这个框架都值得保留：

- **契约（contract）**：什么叫"对"。从被测系统的官方文档导出（PGFuzz 式），定义 oracle。**纪律**：阈值绑定可查参数、看数据前冻结、操作化窗口要对（第十幕的教训：窗口选择能凭空造出/抹掉违例）。
- **输入（input）**：什么叫"合法可发出"。受限位 + 速率约束的飞手机动，定义搜索空间。分类：脉冲回中 / 持续保持 / 多轴协同 / 反向。
- **bug** = ∃ 合法输入，产生的行为违反某契约 = (输入形态 × 契约) 网格里的一格。
- **三者不可分**：能评估哪个契约取决于输入形态；bug 类是网格的格子；控制分配判据结构化每格的通道搜索。
- **PX4 位置模式的文档契约集**（可复用清单）：回中→刹停/回平/锁位；机动→速度上限（MPC_VEL_MANUAL / MPC_XY_VEL_MAX）、倾角（MPC_MAN_TILT_MAX）、加速度（MPC_ACC_HOR_MAX）；另有死区（MPC_HOLD_DZ）、近地限速（MPC_LAND_*）等。

## 6. 方法现状 + 对优秀 ISSTA 论文的诚实差距

**现状一句话**：方法目前不指向一个 bug，而指向一个（待确认的）"PX4 位置模式对可行飞手输入在这组契约下稳健"的负面结论 + 刹车动力学刻画 + 一套自查方法论。

**ISSTA 标杆对照**（"找 bug"论文需要：真实、可复现、最好开发者确认的 bug；对 SOTA 有明确推进；多目标/配置泛化）：

| 维度 | 状态 |
|---|---|
| 问题意义（飞手输入安全） | 够：选题好 |
| 方法新意（控制分配导向、可行输入证伪） | 概念上有，但其宣称优势已被自身实验打掉 |
| 复现 / 严谨性（预注册、复核） | 够，是亮点 |
| 与基线比较（随机、ddmin） | 做了，但若无真 bug 则比较失去意义 |
| **真实、被确认的 bug** | **没有——锚点极可能是假象。致命缺口** |
| 泛化（多平台/模式/种子） | 不够：单平台、单模式、多数单种子；ArduPilot 零证据 |

**结论**：差距不是"补几个种子、润色"的化妆问题，而是**最关键那格是空的**：没有一个经得起复核的违例。没有它，既不成"找 bug 论文"，也不成"方法论文"（方法价值要靠找到真东西证明）。够的只有外围（选题、严谨性、方法论自查）；缺的是核心（一个真 bug，或一个能在真任务上证明的方法优势）。

## 9. 诚实威胁 + 方法论教训

**威胁**：
- 中心对象存在性存疑（首要）；单平台/单模式/多数单种子；ArduPilot 零证据；sim-to-real；判据虽原则化但 RGA 升级不可信（cond 98.65）。

**可迁移的方法论教训**（无论换不换方向都带走）：
1. **尾段窗口混叠**：STL 残余/刹停类契约，若评估窗口从"指令撤除瞬间"起算并整段取峰，会把正常的主动收敛暂态误判为违例。**正确做法是终端窗（或显式分离"暂态"与"稳态残余"）**，并在看数据前固定。
2. **饱和探针是必要不充分**：饱和安全 ≠ 无 bug，对非单调/带动量的契约尤其（但本项目最终发现这里恰恰相反——内部也不违例，是度量假象）。
3. **预注册先提交再跑**：否则只能算 exploratory，确证性主张作废。
4. **复核能救命**：正是"拿已知违例 θ 过一遍新口径"这一步，在投稿前抓住了假象。这套纪律本身是项目最稳的产出。
